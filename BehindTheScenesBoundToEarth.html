Bound to Earth

Case study

Bound to Earth is a 2D platformer where you play as a ghost revisiting key places from his past to unlock memories and determine whether he is a benevolent or malevolent spirit.

The demo focuses on delivering a strong vertical slice centered on exploration, atmosphere and emotional progression.

Arquitectura

The architectural design was focused on beingscalable and modular, using a GameController as the central hub, managing global logic while preventing direct dependencies between systems. This structure allows new modules to be added or removed without affecting the core functionality.

The GameController coordinates: scene transitions, collectibles, post-processing, level progression, events and general gameplay flow.

Core Systems

  • Interaction System – Inspired by previous 3D workflow: IInteractable interface with methods, the player uses Raycast to detect interactive objects. Each object implements its own logic independently from the player.
    Advantages: Highly extensibility, easy to test and easily supporting new interaction types.
  • Player Architecture – A custom 2D Character Controller with:
    - Smooth horizontal acceleration/deceleration
    - Custom jump curve with tailored gravity
    - Reliable ground detection via Raycast/CapsuleCast
    - Clear state separation (Idle, Walk, Fall, Float…)
  • Level Design – The first level introduces core mechanics such as movement, jumping, double jump, interaction and light possession. Although linear, it uses composition and environmental cues to guide players naturally.
  • Collectible-Based Puzzle System – Each level contains hidden memories in the form of collectible fragments. These fragments are stored as Scriptable Objects, allowing each one to define its own data (visual icon, text, lore and unlock conditions) without adding new code.
    Advantages:Fully data-driven workflow. All collected fragments contribute to the Memory Mosaic located in the main hub, a narrative puzzle that slowly reconstructs the ghost’s past across multiple levels.

    • Key Mechanics

    • Precision 2D movement – smooth walking, custom jumping and ghost flying.
    • Exploration – discovering memories, hidden paths and optional fragments.
    • Interaction system (IInteractable) – objects, notes, doors and memory triggers.
    • Collectible memory fragments – used to unlock pieces of the central Memory Mosaic.
    • Environmental storytelling – small scenes and objects that reveal the ghost’s past.
    • Light environmental puzzles – switches, simple logic paths and collectibles placement.
    • Ghost abilities – floating, light possession moments and controlled traversal boosts.

    Development Goals

    The objective of the project was to produce a strong and self-contained vertical slice, demonstrating proficiency in:

    • Systems architecture
    • Modular interaction design
    • Art and animation integration
    • Level design and onboarding
    • Narrative flow and pacing
    • Clean, scalable and maintainable programming

    Code & Best Practices

    • Modular programming and responsibility separation
    • Use of Scriptable Objects as lightweight data containers
    • Event-driven communication to decouple systems
    • Dictionaries for lookup (interactions, items, states)
    • Clean and organized folder structure/li>
