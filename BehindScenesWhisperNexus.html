Whisper of the Nexus

Case study

It was my first large-scale project (demo developed in 2 months). Set in a world engulfed by darkness, the player must free each village by helping its inhabitants return to reality.

The current demo focuses on the first fully functional village, introducing the attack mechanic, occasional support from an NPC, several environmental puzzles, and a final boss encounter. final boss encounter.

Arquitectura

The architectural design was focused on beingscalable and modular, using a GameController as the central core that manages global logic and prevents direct dependencies between systems. It allows adding or removing modules without compromising the overall functionality of the game.

Manages egame states (dialogue, puzzles, global events), coordination between UI and gameplay, and input switching depending on context.

Core Systems

  • Interaction System – Raycast-based + IInteractable interface + InteractableInfo (editable via Inspector).
    Advantages:decoupled, extendable without additional code, compatible with both controller and keyboard.
  • Dialogue System – Dialogue Manager + Scriptable Objects.
    Advantages: Adaptive UI, supports triggers and in-dialogue events, progression options.
  • Puzzle System – Independent script communicating through events.
    Advantages: Clear logic, isolated, easy to maintain.
  • Inventory System – Interactables using their own Scriptable Object.
    Advantages: no need to expand code when adding new items.
  • Player Movement – Blend Tree–based animations, jump with height curve, abilities handled by independent scripts.
  • NPC System –
    Advantages Modular, no script duplication, capable of triggering events and dialogues.
  • Centralized Event System – Coordinates UI, interactions, and logic between systems.

Key Mecanics

  • Exploration and discovery
  • Interaction with objects/NPCs (via IInteractable)
  • Environmental puzzles
  • Resource/flower collection
  • Narrative side quests
  • Light combat system
  • Memory/forgetfulness system as a narrative core

Code & Best Practices

  • Modular programming and responsibility separation
  • Use of Scriptable Objects as lightweight data containers
  • Event-driven communication to decouple systems
  • Dictionaries for lookup (interactions, items, states)
  • Clean and organized folder structure/li>
