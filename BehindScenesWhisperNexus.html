Whisper of the Nexus

Case study

Fue mi primer proyecto grande (demo desarrollada en 2 meses). Ambientado en un mundo sumido en la oscuridad, el jugador debe liberar cada pueblo ayudando a los aldeanos a recuperar la realidad.

La demo actual se centra en el primer pueblo completamente funcional, introduciendo la mecánica de ataque, con ayuda ocasional de un NPC, varios puzles ambientales y un boss final.

Arquitectura

El diseño arquitectónico se centró en ser escalable y modular, con un GameController como núcleo que centraliza toda la lógica y evita dependencias directas entre módulos. Permite agregar o quitar sistemas sin comprometer la funcionalidad global.

Gestiona: estados de juego (diálogo, puzzles, eventos globales), coordinación entre UI y gameplay, y cambios de input según contexto.

Sistemas principales

  • Interaction System – Raycast + interfaz IInteractable + InteractableInfo (editable desde Inspector). Ventajas: desacoplado, extensible sin añadir código, compatible con mando y teclado.
  • Diálogos – Dialogue Manager + Scriptable Objects. Ventajas: UI adaptativa, triggers y eventos durante el diálogo, opciones de progreso.
  • Puzzles – Script independiente con comunicación por eventos. Ventajas: lógica clara, aislada y fácil de mantener.
  • Inventario – Interactuables con SO propio, sin necesidad de expandir código.
  • Player Movement – Animaciones con Blend Tree, salto con curva de altura, poderes mediante scripts independientes.
  • Sistema NPC – Modulares, sin duplicar scripts, capaces de iniciar eventos y diálogos.
  • Sistema centralizado de eventos – Coordina UI, interacciones y lógica entre sistemas.

Mecánicas principales

  • Exploración y descubrimiento
  • Interacción con objetos/NPC (IInteractable)
  • Puzles ambientales
  • Recolección de flores / recursos
  • Misiones narrativas
  • Combate ligero
  • Sistema narrativo de memoria/olvido del personaje

Código y buenas prácticas

  • Programación modular y separación de responsabilidades
  • Uso de Scriptable Objects como base de datos ligera
  • Eventos para desacoplar módulos
  • Diccionarios para lookup (interacciones, items, estados)
  • Estructura clara de carpetas
